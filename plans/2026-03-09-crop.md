### Frontend (deferred — backend first)

## Parallel Execution Plan

- Wave 1: all entity + event files in parallel (no dependencies)
- Wave 2: errors in parallel, then repositories in parallel, then use cases (depend on repos + errors)
- Wave 3: factories + in-memory repos in parallel, then tests (depend on Wave 2)
- Wave 4: schema update → generate → migrate → mappers in parallel → Prisma repos in parallel → database module
- Wave 5: presenters + filter in parallel → CASL files in parallel → controllers (depend on all above) → HTTP module
- Wave 6: event subscribers → event module → register in AppModule
