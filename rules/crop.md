# Crop — Business Rules

## Responsibility

Manage crop types (cultures), varieties, and harvests (safras) — the agricultural cycles that happen on fields.

## Out of scope

- Field management (owned by Field domain)
- Operation planning and scheduling (owned by Schedule domain)
- Financial tracking per harvest (owned by Financial domain)
- Input/inventory management (owned by Inventory domain)

## Entities

### CropType (Cultura)

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| name | string | yes | unique, accent-insensitive comparison |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name must be unique across all active crop types (accent-insensitive: "Vitória" and "Vitoria" are the same)
- Cannot hard delete if any Variety or Harvest references this CropType
- Soft delete when referenced by existing harvests

**Lifecycle:** no state machine — active or soft-deleted

---

### Variety (Variedade)

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| cropTypeId | UUID | yes | references CropType |
| name | string | yes | unique per CropType, accent-insensitive comparison |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name must be unique within the same CropType (accent-insensitive: "Vitória" and "Vitoria" are the same)
- The same name can exist in different CropTypes
- Cannot hard delete if any Harvest references this Variety
- Soft delete when referenced by existing harvests

**Lifecycle:** no state machine — active or soft-deleted

---

### Harvest (Safra)

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| cropTypeId | UUID | yes | references CropType |
| varietyId | UUID | yes | references Variety (must belong to the same CropType) |
| fieldId | UUID | yes | references Field (external domain, by ID only) |
| startDate | DateTime | yes | must be a valid date |
| expectedEndDate | DateTime | yes | must be after startDate |
| actualEndDate | DateTime | no | set when harvest is completed |
| status | enum | yes | UNSCHEDULED, ACTIVE, COMPLETED, CANCELLED |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- A field can only have one active harvest at a time (status UNSCHEDULED or ACTIVE)
- expectedEndDate must be after startDate
- varietyId must belong to the referenced cropTypeId
- Cannot cancel or complete if there are operations in progress (schedules with active field tickets)
- Cannot hard delete if schedules or field tickets reference this harvest
- Soft delete when referenced by completed operations

**Lifecycle:**

```
UNSCHEDULED → ACTIVE (via ScheduleActivatedEvent, not manual)
UNSCHEDULED → CANCELLED
ACTIVE → COMPLETED (manual, Schedule must be COMPLETED)
ACTIVE → CANCELLED
```

- UNSCHEDULED: harvest created, no schedule yet
- ACTIVE: harvest in progress (activated via Schedule domain)
- COMPLETED: harvest finished (actualEndDate is set)
- CANCELLED: harvest abandoned (only if no operations in progress)

## Value Objects

None — all attributes are primitive types for now.

## Use Cases

### CreateCropType
- **Trigger:** user action (form submission)
- **Input:** name
- **Rules:** name must be unique (accent-insensitive). If user submits "Vitoria" and "Vitória" already exists, reject with error suggesting the existing name.
- **Success:** CropType created
- **Errors:** CropTypeAlreadyExistsError (includes the existing correctly-accented name)
- **Events emitted:** CropTypeCreatedEvent

### EditCropType
- **Trigger:** user action
- **Input:** cropTypeId, name
- **Rules:** new name must be unique (accent-insensitive), excluding self
- **Success:** CropType updated
- **Errors:** CropTypeNotFoundError, CropTypeAlreadyExistsError
- **Events emitted:** CropTypeUpdatedEvent

### DeleteCropType
- **Trigger:** user action
- **Input:** cropTypeId
- **Rules:** hard delete if no varieties or harvests reference it. Soft delete if referenced.
- **Success:** CropType deleted (hard or soft)
- **Errors:** CropTypeNotFoundError, CropTypeHasActiveHarvestsError (if any harvest is PLANNED or ACTIVE)
- **Events emitted:** CropTypeDeletedEvent

### ListCropTypes
- **Trigger:** user navigates to crop types page
- **Input:** pagination params, optional search filter
- **Rules:** only return non-soft-deleted records
- **Success:** paginated list of crop types
- **Errors:** none
- **Events emitted:** none

### CreateVariety
- **Trigger:** user action
- **Input:** cropTypeId, name
- **Rules:** name must be unique within the CropType (accent-insensitive)
- **Success:** Variety created
- **Errors:** CropTypeNotFoundError, VarietyAlreadyExistsError
- **Events emitted:** VarietyCreatedEvent

### EditVariety
- **Trigger:** user action
- **Input:** varietyId, name
- **Rules:** new name must be unique within the same CropType (accent-insensitive), excluding self
- **Success:** Variety updated
- **Errors:** VarietyNotFoundError, VarietyAlreadyExistsError
- **Events emitted:** VarietyUpdatedEvent

### DeleteVariety
- **Trigger:** user action
- **Input:** varietyId
- **Rules:** hard delete if no harvests reference it. Soft delete if referenced.
- **Success:** Variety deleted (hard or soft)
- **Errors:** VarietyNotFoundError, VarietyHasActiveHarvestsError
- **Events emitted:** VarietyDeletedEvent

### ListVarieties
- **Trigger:** user action or select dropdown
- **Input:** cropTypeId, pagination params
- **Rules:** only return non-soft-deleted records for the given CropType
- **Success:** paginated list of varieties
- **Errors:** CropTypeNotFoundError
- **Events emitted:** none

### CreateHarvest
- **Trigger:** user action
- **Input:** cropTypeId, varietyId, fieldId, startDate, expectedEndDate
- **Rules:** field must not have an active harvest (PLANNED or ACTIVE). Variety must belong to the CropType. expectedEndDate must be after startDate.
- **Success:** Harvest created with status PLANNED
- **Errors:** FieldAlreadyHasActiveHarvestError, VarietyDoesNotBelongToCropTypeError, InvalidDateRangeError
- **Events emitted:** HarvestCreatedEvent

### EditHarvest
- **Trigger:** user action
- **Input:** harvestId, startDate, expectedEndDate (cropType/variety/field are immutable after creation)
- **Rules:** expectedEndDate must be after startDate. Cannot edit if COMPLETED or CANCELLED.
- **Success:** Harvest updated
- **Errors:** HarvestNotFoundError, InvalidDateRangeError, HarvestNotEditableError
- **Events emitted:** HarvestUpdatedEvent

### ActivateHarvest
- **Trigger:** ScheduleActivatedEvent (not manual — no HTTP endpoint)
- **Input:** harvestId, actorId (from event)
- **Rules:** status must be UNSCHEDULED.
- **Success:** Harvest status → ACTIVE
- **Errors:** HarvestNotFoundError, HarvestNotUnscheduledError
- **Events emitted:** HarvestActivatedEvent

### CompleteHarvest
- **Trigger:** user action
- **Input:** harvestId
- **Rules:** status must be ACTIVE. Must not have operations in progress (active schedules/field tickets). Sets actualEndDate to now.
- **Success:** Harvest status → COMPLETED
- **Errors:** HarvestNotFoundError, HarvestNotActiveError, HarvestHasOperationsInProgressError
- **Events emitted:** HarvestCompletedEvent

### CancelHarvest
- **Trigger:** user action
- **Input:** harvestId
- **Rules:** status must be PLANNED or ACTIVE. Must not have operations in progress.
- **Success:** Harvest status → CANCELLED
- **Errors:** HarvestNotFoundError, HarvestNotCancellableError, HarvestHasOperationsInProgressError
- **Events emitted:** HarvestCancelledEvent

### DeleteHarvest
- **Trigger:** user action
- **Input:** harvestId
- **Rules:** hard delete if no schedules or field tickets reference it. Soft delete if referenced.
- **Success:** Harvest deleted (hard or soft)
- **Errors:** HarvestNotFoundError, HarvestHasActiveOperationsError
- **Events emitted:** HarvestDeletedEvent

### GetActiveHarvestByField
- **Trigger:** other domains querying (Schedule needs this)
- **Input:** fieldId
- **Rules:** return harvest with status PLANNED or ACTIVE for the given field
- **Success:** Harvest or null
- **Errors:** none
- **Events emitted:** none

### ListHarvests
- **Trigger:** user navigates to harvests page
- **Input:** pagination params, optional filters (fieldId, cropTypeId, status)
- **Rules:** only return non-soft-deleted records
- **Success:** paginated list of harvests
- **Errors:** none
- **Events emitted:** none

## Constraints

- CropType name: unique, accent-insensitive (normalize with Unicode NFD + strip diacritics for comparison)
- Variety name: unique per CropType, accent-insensitive
- One active harvest per field (status UNSCHEDULED or ACTIVE)
- Variety must belong to the referenced CropType
- All operations audited
- Soft delete preserves data for audit trail and future reports
- Hard delete only when no downstream dependencies exist

## Edge Cases

- User submits "Vitoria" when "Vitória" exists → reject with error suggesting "Vitória"
- User tries to create harvest on a field that already has an UNSCHEDULED harvest → reject
- User tries to cancel active harvest with field tickets in progress → reject
- User deletes CropType that has only soft-deleted varieties → allowed (hard delete)
- User deletes CropType that has active varieties → soft delete CropType
- Concurrent harvest creation on the same field → database unique constraint prevents duplicates

## Resolved Questions

- **UNSCHEDULED → ACTIVE transition:** Automatic — driven by ScheduleActivatedEvent when a Schedule is activated. No manual activation endpoint.
- **Harvest cancellation cascading:** Yes — cancelling a harvest automatically cancels its schedules via domain event (HarvestCancelledEvent). All records are preserved (soft delete) for audit trail and reports.
- **Schedule cancelled → Harvest reverts:** When a Schedule is cancelled, the Harvest reverts to UNSCHEDULED, allowing a new Schedule to be created.
