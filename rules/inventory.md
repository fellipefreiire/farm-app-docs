# Inventory — Business Rules

## Responsibility

Manage agricultural input stock — categories, input registry, purchase entries, and stock exits (manual and automatic).

## Out of scope

- Lot/batch tracking and expiry dates (post-MVP)
- Minimum stock alerts (post-MVP)
- Soft reservation for Schedule planning (post-MVP)
- Automatic stock exit via FieldTicket finalization (MVP — implemented when FieldTicket domain is built)
- Supplier management (handled by Supplier domain)
- Financial reporting on purchases (handled by Financial domain, post-MVP)

---

## Entities

### Category

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| name | string | yes | unique, accent-insensitive comparison |
| status | enum | yes | ACTIVE or INACTIVE, defaults to ACTIVE |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name must be unique across all active (non-soft-deleted) categories (accent-insensitive: "Herbicida" and "herbicida" are the same)
- Cannot hard delete if any Input references this Category
- Soft delete when referenced by existing inputs

**Lifecycle:** ACTIVE ↔ INACTIVE (toggle). Soft-deleted when referenced and user requests deletion.

---

### Input

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| name | string | yes | unique per category, accent-insensitive comparison |
| categoryId | UUID | yes | references Category |
| unitOfMeasure | enum | yes | L, kg, mL, g, un |
| status | enum | yes | ACTIVE or INACTIVE, defaults to ACTIVE |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name must be unique within the same Category (accent-insensitive: "Produto X" and "produto x" are the same within one category)
- The same name can exist in different Categories
- categoryId must reference an existing, active Category
- Cannot hard delete if any PurchaseItem or StockMovement references this Input
- Soft delete when referenced by existing purchases or stock movements

**Lifecycle:** ACTIVE ↔ INACTIVE (toggle). Soft-deleted when referenced and user requests deletion.

---

### Purchase

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| supplierId | UUID | yes | references Supplier (cross-domain, by ID) |
| purchaseDate | DateTime | yes | date of the purchase |
| items | PurchaseItem[] | yes | at least one item required |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |

**Invariants:**
- Must have at least one PurchaseItem
- supplierId must reference an existing, active Supplier
- Cannot be deleted — only items can be adjusted (audit trail preservation)

**Lifecycle:** no state machine

---

### PurchaseItem

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| purchaseId | UUID | yes | references Purchase |
| inputId | UUID | yes | references Input |
| quantity | decimal | yes | must be greater than 0 |
| totalValue | decimal | yes | must be greater than or equal to 0, in BRL |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |

**Invariants:**
- quantity must be > 0
- totalValue must be >= 0
- inputId must reference an existing Input
- Quantity unit is determined by the referenced Input's unitOfMeasure

**Lifecycle:** no state machine

---

### StockMovement

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| inputId | UUID | yes | references Input |
| quantity | decimal | yes | must be greater than 0 |
| type | enum | yes | MANUAL or AUTOMATIC (AUTOMATIC implemented when FieldTicket domain is built) |
| reason | enum | yes (when MANUAL) | LOSS, DISCARD, ADJUSTMENT, INTERNAL_USE |
| fieldTicketId | UUID | no | references FieldTicket (future — when type is AUTOMATIC) |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |

**Invariants:**
- quantity must be > 0
- When type is MANUAL, reason is required
- When type is AUTOMATIC, fieldTicketId is required (future)
- inputId must reference an existing Input
- Stock balance (sum of entries - sum of exits) can go negative (no blocking — real world may have discrepancies)

**Lifecycle:** no state machine

---

## Value Objects

- **UnitOfMeasure** — enum: L (liters), kg (kilograms), mL (milliliters), g (grams), un (units). Fixed in the system, not user-configurable.
- **StockMovementType** — enum: MANUAL, AUTOMATIC. AUTOMATIC implemented when FieldTicket domain is built.
- **StockMovementReason** — enum: LOSS, DISCARD, ADJUSTMENT, INTERNAL_USE.
- **Money** — decimal, BRL only. Used for PurchaseItem.totalValue. Must be >= 0.

## Use Cases

### CreateCategory
- **Trigger:** user action
- **Input:** name
- **Rules:** name must be unique (accent-insensitive)
- **Success:** Category entity created
- **Errors:** CategoryAlreadyExistsError
- **Events emitted:** CategoryCreatedEvent

### EditCategory
- **Trigger:** user action
- **Input:** categoryId, name
- **Rules:** if name changed, new name must be unique (accent-insensitive)
- **Success:** Category entity updated
- **Errors:** CategoryNotFoundError, CategoryAlreadyExistsError
- **Events emitted:** CategoryUpdatedEvent

### DeleteCategory
- **Trigger:** user action
- **Input:** categoryId
- **Rules:** hard delete if no inputs reference it. Soft delete if referenced.
- **Success:** Category deleted (hard or soft)
- **Errors:** CategoryNotFoundError
- **Events emitted:** CategoryDeletedEvent

### ToggleCategoryStatus
- **Trigger:** user action
- **Input:** categoryId
- **Rules:** toggles status between ACTIVE and INACTIVE
- **Success:** Category status toggled
- **Errors:** CategoryNotFoundError
- **Events emitted:** CategoryUpdatedEvent

### ListCategories
- **Trigger:** user action or select dropdown (e.g., Input form)
- **Input:** pagination params (page, perPage), optional filters (status, search by name)
- **Rules:** only return non-soft-deleted records, paginated
- **Success:** paginated list of Category entities
- **Errors:** none
- **Events emitted:** none

---

### CreateInput
- **Trigger:** user action
- **Input:** name, categoryId, unitOfMeasure
- **Rules:** name must be unique within the category (accent-insensitive). Category must exist and be active.
- **Success:** Input entity created
- **Errors:** InputAlreadyExistsError, CategoryNotFoundError
- **Events emitted:** InputCreatedEvent

### EditInput
- **Trigger:** user action
- **Input:** inputId, name (optional), categoryId (optional), unitOfMeasure (optional)
- **Rules:** if name or category changed, uniqueness check within new/existing category (accent-insensitive). New category must exist and be active.
- **Success:** Input entity updated
- **Errors:** InputNotFoundError, InputAlreadyExistsError, CategoryNotFoundError
- **Events emitted:** InputUpdatedEvent

### DeleteInput
- **Trigger:** user action
- **Input:** inputId
- **Rules:** hard delete if no purchase items or stock movements reference it. Soft delete if referenced.
- **Success:** Input deleted (hard or soft)
- **Errors:** InputNotFoundError
- **Events emitted:** InputDeletedEvent

### ToggleInputStatus
- **Trigger:** user action
- **Input:** inputId
- **Rules:** toggles status between ACTIVE and INACTIVE
- **Success:** Input status toggled
- **Errors:** InputNotFoundError
- **Events emitted:** InputUpdatedEvent

### GetInput
- **Trigger:** user action
- **Input:** inputId
- **Rules:** only return non-soft-deleted records
- **Success:** Input entity with current stock balance (calculated)
- **Errors:** InputNotFoundError
- **Events emitted:** none

### ListInputs
- **Trigger:** user action or select dropdown (e.g., Purchase form)
- **Input:** pagination params (page, perPage), optional filters (categoryId, status, search by name)
- **Rules:** only return non-soft-deleted records, paginated. Include current stock balance per input.
- **Success:** paginated list of Input entities with stock balances
- **Errors:** none
- **Events emitted:** none

---

### CreatePurchase
- **Trigger:** user action
- **Input:** supplierId, purchaseDate, items (array of: inputId, quantity, totalValue)
- **Rules:** supplier must exist and be active. Each inputId must exist. At least one item required. Quantity > 0, totalValue >= 0 for each item.
- **Success:** Purchase entity created with all items. Stock balances updated (calculated).
- **Errors:** SupplierNotFoundError, InputNotFoundError, EmptyPurchaseItemsError
- **Events emitted:** PurchaseCreatedEvent

### EditPurchase
- **Trigger:** user action
- **Input:** purchaseId, editReason (string), supplierId (optional), purchaseDate (optional), items (optional — full replacement of items list)
- **Rules:** editReason is required (describes why the purchase was modified). Same validation as CreatePurchase for changed fields. If items are replaced, recalculates stock balances.
- **Success:** Purchase entity updated. Stock balances recalculated.
- **Errors:** PurchaseNotFoundError, SupplierNotFoundError, InputNotFoundError, EmptyPurchaseItemsError
- **Events emitted:** PurchaseUpdatedEvent

### GetPurchase
- **Trigger:** user action
- **Input:** purchaseId
- **Rules:** none
- **Success:** Purchase entity with all items, supplier info, and input names
- **Errors:** PurchaseNotFoundError
- **Events emitted:** none

### ListPurchases
- **Trigger:** user action
- **Input:** pagination params (page, perPage), optional filters (supplierId, date range, inputId)
- **Rules:** paginated, ordered by purchaseDate descending
- **Success:** paginated list of Purchases with supplier name and item count
- **Errors:** none
- **Events emitted:** none

---

### CreateStockMovement
- **Trigger:** user action (manual exit)
- **Input:** inputId, quantity, reason (LOSS, DISCARD, ADJUSTMENT, INTERNAL_USE)
- **Rules:** input must exist. Quantity > 0. Reason required. Stock balance can go negative (no blocking).
- **Success:** StockMovement entity created. Stock balance updated (calculated).
- **Errors:** InputNotFoundError
- **Events emitted:** StockMovementCreatedEvent

### EditStockMovement
- **Trigger:** user action
- **Input:** stockMovementId, inputId (optional), quantity (optional), reason (optional)
- **Rules:** same validation as create for changed fields
- **Success:** StockMovement entity updated. Stock balance recalculated.
- **Errors:** StockMovementNotFoundError, InputNotFoundError
- **Events emitted:** StockMovementUpdatedEvent

### CancelStockMovement
- **Trigger:** user action (undo accidental exit)
- **Input:** stockMovementId
- **Rules:** marks movement as cancelled, reverses stock impact
- **Success:** StockMovement cancelled. Stock balance recalculated.
- **Errors:** StockMovementNotFoundError, StockMovementAlreadyCancelledError
- **Events emitted:** StockMovementCancelledEvent

### ListStockMovements
- **Trigger:** user action
- **Input:** pagination params (page, perPage), optional filters (inputId, type, reason, date range)
- **Rules:** paginated, ordered by createdAt descending
- **Success:** paginated list of StockMovements with input name
- **Errors:** none
- **Events emitted:** none

---

### GetStockBalance
- **Trigger:** user action, or other domains querying
- **Input:** inputId
- **Rules:** calculated as sum(PurchaseItem.quantity for this input) - sum(active StockMovement.quantity for this input)
- **Success:** { inputId, inputName, unitOfMeasure, balance }
- **Errors:** InputNotFoundError
- **Events emitted:** none

## Constraints

- Category name uniqueness across all active (non-soft-deleted) categories, accent-insensitive
- Input name uniqueness within same category among active (non-soft-deleted) inputs, accent-insensitive
- Soft delete preserves data for audit trail and historical references
- Hard delete only when no downstream dependencies exist
- All mutation operations audited via domain events
- Stock balance is always calculated, never stored as a field
- Purchase cannot be deleted — only edited with reason (audit trail)
- Inactive inputs/categories visible in existing records but cannot be selected for new purchases or movements

## Edge Cases

- User submits "Herbicida" when "herbicida" exists as category → reject with error suggesting existing name
- User creates Input "Produto X" in category "Herbicida" when "Produto X" already exists in "Fungicida" → allowed (uniqueness is per category)
- User creates Input "Produto X" in category "Herbicida" when "produto x" already exists in "Herbicida" → reject
- User deletes Category that has only soft-deleted inputs → allowed (hard delete)
- User deletes Category that has active inputs → soft delete
- User edits Purchase changing quantity → stock balance automatically recalculated
- Stock balance goes negative after a StockMovement → allowed (no blocking — real discrepancies happen)
- User cancels a StockMovement → stock balance recalculated (quantity returned)
- Inactive Input shown in existing Purchase records but cannot be selected for new purchases
- Inactive Category shown in existing Input records but cannot be selected when creating new inputs
- User creates Purchase with an inactive Supplier → rejected (supplier must be active)
- User edits Purchase with editReason empty → rejected (reason is mandatory for audit trail)
