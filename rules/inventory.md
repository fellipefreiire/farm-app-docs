# Inventory — Business Rules

## Responsibility

Manage agricultural input stock — categories, input registry, purchase entries, stock exits (manual and automatic).

## Out of scope

- Lot/batch tracking + expiry dates (post-MVP)
- Min stock alerts (post-MVP)
- Soft reservation for Schedule planning (post-MVP)
- Auto stock exit via FieldTicket finalization (MVP — when FieldTicket domain built)
- Supplier management (Supplier domain)
- Financial reporting on purchases (Financial domain, post-MVP)

---

## Entities

### Category

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| name | string | yes | unique, accent-insensitive comparison |
| status | enum | yes | ACTIVE or INACTIVE, defaults to ACTIVE |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name unique across all active (non-soft-deleted) categories (accent-insensitive: "Herbicida" = "herbicida")
- No hard delete if any Input references Category
- Soft delete when referenced by existing inputs

**Lifecycle:** ACTIVE ↔ INACTIVE (toggle). Soft-deleted when referenced + user requests deletion.

---

### Input

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| name | string | yes | unique per category, accent-insensitive comparison |
| categoryId | UUID | yes | references Category |
| unitOfMeasure | enum | yes | L, kg, mL, g, un |
| status | enum | yes | ACTIVE or INACTIVE, defaults to ACTIVE |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name unique within same Category (accent-insensitive: "Produto X" = "produto x" within one category)
- Same name allowed in different Categories
- `categoryId` must reference existing, active Category
- No hard delete if any PurchaseItem or StockMovement references Input
- Soft delete when referenced by existing purchases or movements

**Lifecycle:** ACTIVE ↔ INACTIVE (toggle). Soft-deleted when referenced + user requests deletion.

---

### Purchase

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| supplierId | UUID | yes | references Supplier (cross-domain, by ID) |
| purchaseDate | DateTime | yes | date of purchase |
| items | PurchaseItem[] | yes | at least one required |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |

**Invariants:**
- Must have ≥1 PurchaseItem
- `supplierId` must reference existing, active Supplier
- Cannot delete — only edit with reason (audit trail)

**Lifecycle:** no state machine

---

### PurchaseItem

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| purchaseId | UUID | yes | references Purchase |
| inputId | UUID | yes | references Input |
| quantity | decimal | yes | > 0 |
| totalValue | decimal | yes | >= 0, BRL |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |

**Invariants:**
- `quantity` > 0
- `totalValue` >= 0
- `inputId` must reference existing Input
- Quantity unit from referenced Input's `unitOfMeasure`

**Lifecycle:** no state machine

---

### StockMovement

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| inputId | UUID | yes | references Input |
| quantity | decimal | yes | > 0 |
| type | enum | yes | MANUAL or AUTOMATIC (AUTOMATIC when FieldTicket domain built) |
| reason | enum | yes (when MANUAL) | LOSS, DISCARD, ADJUSTMENT, INTERNAL_USE |
| fieldTicketId | UUID | no | references FieldTicket (future — when type is AUTOMATIC) |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |

**Invariants:**
- `quantity` > 0
- MANUAL → reason required
- AUTOMATIC → `fieldTicketId` required (future)
- `inputId` must reference existing Input
- Balance (entries - exits) can go negative (no blocking — real discrepancies happen)

**Lifecycle:** no state machine

---

## Value Objects

- **UnitOfMeasure** — enum: L, kg, mL, g, un. Fixed, not user-configurable.
- **StockMovementType** — enum: MANUAL, AUTOMATIC. AUTOMATIC when FieldTicket domain built.
- **StockMovementReason** — enum: LOSS, DISCARD, ADJUSTMENT, INTERNAL_USE.
- **Money** — decimal, BRL only. Used for `PurchaseItem.totalValue`. Must be >= 0.

## Use Cases

### CreateCategory
- **Trigger:** user action
- **Input:** name
- **Rules:** name unique (accent-insensitive)
- **Success:** Category created
- **Errors:** CategoryAlreadyExistsError
- **Events emitted:** CategoryCreatedEvent

### EditCategory
- **Trigger:** user action
- **Input:** categoryId, name
- **Rules:** if name changed, new name must be unique (accent-insensitive)
- **Success:** Category updated
- **Errors:** CategoryNotFoundError, CategoryAlreadyExistsError
- **Events emitted:** CategoryUpdatedEvent

### DeleteCategory
- **Trigger:** user action
- **Input:** categoryId
- **Rules:** hard delete if no inputs reference it; soft delete if referenced
- **Success:** Category deleted (hard or soft)
- **Errors:** CategoryNotFoundError
- **Events emitted:** CategoryDeletedEvent

### ToggleCategoryStatus
- **Trigger:** user action
- **Input:** categoryId
- **Rules:** toggles ACTIVE ↔ INACTIVE
- **Success:** Category status toggled
- **Errors:** CategoryNotFoundError
- **Events emitted:** CategoryUpdatedEvent

### ListCategories
- **Trigger:** user action or select dropdown (e.g., Input form)
- **Input:** pagination (page, perPage), optional filters (status, name search)
- **Rules:** non-soft-deleted only, paginated
- **Success:** paginated Category list
- **Errors:** none
- **Events emitted:** none

---

### CreateInput
- **Trigger:** user action
- **Input:** name, categoryId, unitOfMeasure
- **Rules:** name unique within category (accent-insensitive). Category must exist + be active.
- **Success:** Input created
- **Errors:** InputAlreadyExistsError, CategoryNotFoundError
- **Events emitted:** InputCreatedEvent

### EditInput
- **Trigger:** user action
- **Input:** inputId, name (optional), categoryId (optional), unitOfMeasure (optional)
- **Rules:** if name/category changed, uniqueness check within new/existing category (accent-insensitive). New category must exist + be active.
- **Success:** Input updated
- **Errors:** InputNotFoundError, InputAlreadyExistsError, CategoryNotFoundError
- **Events emitted:** InputUpdatedEvent

### DeleteInput
- **Trigger:** user action
- **Input:** inputId
- **Rules:** hard delete if no purchase items or movements reference it; soft delete if referenced
- **Success:** Input deleted (hard or soft)
- **Errors:** InputNotFoundError
- **Events emitted:** InputDeletedEvent

### ToggleInputStatus
- **Trigger:** user action
- **Input:** inputId
- **Rules:** toggles ACTIVE ↔ INACTIVE
- **Success:** Input status toggled
- **Errors:** InputNotFoundError
- **Events emitted:** InputUpdatedEvent

### GetInput
- **Trigger:** user action
- **Input:** inputId
- **Rules:** non-soft-deleted only
- **Success:** Input with current stock balance (calculated)
- **Errors:** InputNotFoundError
- **Events emitted:** none

### ListInputs
- **Trigger:** user action or select dropdown (e.g., Purchase form)
- **Input:** pagination (page, perPage), optional filters (categoryId, status, name search)
- **Rules:** non-soft-deleted only, paginated. Include stock balance per input.
- **Success:** paginated Input list with stock balances
- **Errors:** none
- **Events emitted:** none

---

### CreatePurchase
- **Trigger:** user action
- **Input:** supplierId, purchaseDate, items (array of: inputId, quantity, totalValue)
- **Rules:** supplier active. Each inputId must exist. ≥1 item. quantity > 0, totalValue >= 0 per item.
- **Success:** Purchase + items created. Stock balances updated (calculated).
- **Errors:** SupplierNotFoundError, InputNotFoundError, EmptyPurchaseItemsError
- **Events emitted:** PurchaseCreatedEvent

### EditPurchase
- **Trigger:** user action
- **Input:** purchaseId, editReason (string), supplierId (optional), purchaseDate (optional), items (optional — full replacement)
- **Rules:** editReason required. Same validation as CreatePurchase for changed fields. Items replaced → recalculate stock balances.
- **Success:** Purchase updated. Stock balances recalculated.
- **Errors:** PurchaseNotFoundError, SupplierNotFoundError, InputNotFoundError, EmptyPurchaseItemsError
- **Events emitted:** PurchaseUpdatedEvent

### GetPurchase
- **Trigger:** user action
- **Input:** purchaseId
- **Rules:** none
- **Success:** Purchase with all items, supplier info, input names
- **Errors:** PurchaseNotFoundError
- **Events emitted:** none

### ListPurchases
- **Trigger:** user action
- **Input:** pagination (page, perPage), optional filters (supplierId, date range, inputId)
- **Rules:** paginated, ordered by purchaseDate desc
- **Success:** paginated Purchases with supplier name + item count
- **Errors:** none
- **Events emitted:** none

---

### CreateStockMovement
- **Trigger:** user action (manual exit)
- **Input:** inputId, quantity, reason (LOSS, DISCARD, ADJUSTMENT, INTERNAL_USE)
- **Rules:** input must exist. quantity > 0. reason required. Balance can go negative (no blocking).
- **Success:** StockMovement created. Stock balance updated (calculated).
- **Errors:** InputNotFoundError
- **Events emitted:** StockMovementCreatedEvent

### EditStockMovement
- **Trigger:** user action
- **Input:** stockMovementId, inputId (optional), quantity (optional), reason (optional)
- **Rules:** same validation as create for changed fields
- **Success:** StockMovement updated. Stock balance recalculated.
- **Errors:** StockMovementNotFoundError, InputNotFoundError
- **Events emitted:** StockMovementUpdatedEvent

### CancelStockMovement
- **Trigger:** user action (undo accidental exit)
- **Input:** stockMovementId
- **Rules:** marks cancelled, reverses stock impact
- **Success:** StockMovement cancelled. Stock balance recalculated.
- **Errors:** StockMovementNotFoundError, StockMovementAlreadyCancelledError
- **Events emitted:** StockMovementCancelledEvent

### ListStockMovements
- **Trigger:** user action
- **Input:** pagination (page, perPage), optional filters (inputId, type, reason, date range)
- **Rules:** paginated, ordered by createdAt desc
- **Success:** paginated StockMovements with input name
- **Errors:** none
- **Events emitted:** none

---

### GetStockBalance
- **Trigger:** user action or other domains querying
- **Input:** inputId
- **Rules:** calculated as sum(PurchaseItem.quantity for input) - sum(active StockMovement.quantity for input)
- **Success:** { inputId, inputName, unitOfMeasure, balance }
- **Errors:** InputNotFoundError
- **Events emitted:** none

## Constraints

- Category name unique across active (non-soft-deleted) categories, accent-insensitive
- Input name unique within same category among active (non-soft-deleted) inputs, accent-insensitive
- Soft delete preserves data for audit trail + historical references
- Hard delete only when no downstream deps exist
- All mutations audited via domain events
- Stock balance always calculated, never stored
- Purchase cannot be deleted — only edited with reason (audit trail)
- Inactive inputs/categories visible in existing records but blocked from new purchases/movements

## Edge Cases

- User submits "Herbicida" when "herbicida" exists → reject, suggest existing name
- User creates "Produto X" in "Herbicida" when "Produto X" exists in "Fungicida" → allowed (uniqueness per category)
- User creates "Produto X" in "Herbicida" when "produto x" exists in "Herbicida" → reject
- User deletes Category with only soft-deleted inputs → allowed (hard delete)
- User deletes Category with active inputs → soft delete
- User edits Purchase changing quantity → stock balance auto-recalculated
- Stock balance goes negative after StockMovement → allowed (no blocking)
- User cancels StockMovement → stock balance recalculated (quantity returned)
- Inactive Input shown in existing Purchase records but blocked from new purchases
- Inactive Category shown in existing Input records but blocked when creating new inputs
- User creates Purchase with inactive Supplier → rejected (supplier must be active)
- User edits Purchase with empty editReason → rejected (mandatory for audit trail)
