# Field — Business Rules

## Responsibility

Manage the physical plots (talhões) of the farm — registration of name, area, and status.

## Out of scope

- Crop planning or planting (owned by Crop/Schedule domains)
- Operations executed on the field (owned by their respective domains)
- Financial tracking per field (owned by Financial domain)
- Multi-farm support (future — Field currently exists independently without farm association)

## Entities

### Field (Talhão)

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| name | string | yes | unique across all active fields |
| area | number | yes | in hectares, must be > 0 |
| status | enum | yes | ACTIVE or INACTIVE, defaults to ACTIVE |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name must be unique across all active fields
- Area must be greater than 0
- Cannot hard delete if any entity from another domain references this Field
- Soft delete when referenced by other domains

**Lifecycle:** ACTIVE ↔ INACTIVE (toggle). Soft-deleted when referenced and user requests deletion.

## Value Objects

None — all attributes are primitives with simple validation.

## Use Cases

### CreateField
- **Trigger:** user action (ADMIN or MANAGER)
- **Input:** name, area
- **Rules:** name must be unique, area > 0, status defaults to ACTIVE
- **Success:** created Field entity
- **Errors:** FieldAlreadyExistsError (duplicate name), InvalidAreaError (area ≤ 0)
- **Events emitted:** FieldCreatedEvent

### EditField
- **Trigger:** user action (ADMIN or MANAGER)
- **Input:** fieldId, name?, area?
- **Rules:** if name changed, must remain unique; if area changed, must be > 0
- **Success:** updated Field entity
- **Errors:** FieldNotFoundError, FieldAlreadyExistsError (duplicate name), InvalidAreaError (area ≤ 0)
- **Events emitted:** FieldUpdatedEvent

### ToggleFieldStatus
- **Trigger:** user action (ADMIN or MANAGER)
- **Input:** fieldId
- **Rules:** toggles between ACTIVE and INACTIVE
- **Success:** updated Field entity with toggled status
- **Errors:** FieldNotFoundError
- **Events emitted:** FieldUpdatedEvent

### DeleteField
- **Trigger:** user action (ADMIN only)
- **Input:** fieldId
- **Rules:** hard delete if no external references exist. Soft delete if referenced by other domains.
- **Success:** field deleted (hard or soft)
- **Errors:** FieldNotFoundError
- **Events emitted:** FieldDeletedEvent (soft delete only)

### GetField
- **Trigger:** user action
- **Input:** fieldId
- **Rules:** only return non-soft-deleted records
- **Success:** Field entity
- **Errors:** FieldNotFoundError
- **Events emitted:** none

### ListFields
- **Trigger:** user action
- **Input:** pagination params (page, perPage), optional filters (status, search by name)
- **Rules:** only return non-soft-deleted records, paginated
- **Success:** paginated list of Field entities
- **Errors:** none
- **Events emitted:** none

## Cross-Domain Dependencies

- **Depends on:** none — Field is an independent domain
- **Referenced by:** other domains reference Field by `fieldId` (e.g., Crop/Harvest, Schedule, Operations)
- **Detail page aggregation:** the Field detail page in the frontend will query other domains by `fieldId` to display active operations on the field. This is a frontend concern, not a domain dependency.

## Constraints

- Name uniqueness across all active (non-soft-deleted) fields
- Area must be a positive number (> 0)
- Soft delete preserves data for audit trail and reports
- Hard delete only when no downstream dependencies exist
- Soft-deleted fields remain visible in other modules that reference them (historical data preserved), but cannot be accessed directly or selected for new records

## Edge Cases

- User creates Field with a name that matches a soft-deleted Field — allowed (uniqueness only applies to active records)
- User deletes Field that has active references in other domains — soft delete
- User deletes Field with no references — hard delete
- Other modules displaying a soft-deleted Field — show field data (name, area) but no link to detail page
- Editing area of a Field with active operations — allowed (other modules reference by ID, not by area value)
