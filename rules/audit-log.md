# Audit Log — Business Rules

## Responsibility
Records an immutable trail of domain events for compliance, debugging, and user activity tracking.

## Out of scope
- Triggering business logic based on audit entries (audit-log is append-only, read-only)
- User management (handled by `user` domain)
- Log rotation or archival (infrastructure concern)

## Entities

### AuditLog (Entity)
| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| actorId | string | yes | ID of the user who performed the action |
| action | string | yes | namespaced action identifier (e.g. `user:created`) |
| entity | string | yes | entity type affected (e.g. `user`) |
| entityId | string | yes | ID of the affected entity |
| changes | Record | no | snapshot of relevant data at the time of the event |
| createdAt | Date | yes | set on creation, never updated |

**Invariants:**
- Audit logs are immutable — once created, they are never updated or deleted
- Every audit log must have an actorId — anonymous actions are not allowed

**Lifecycle:** No state machine — created once, never modified.

## Value Objects
None.

## Use Cases

### CreateAuditLog
- **Trigger:** Domain event subscriber receives an event (not called directly by controllers)
- **Input:** actorId, action, entity, entityId, changes (optional)
- **Rules:** None — all valid events are logged
- **Success:** `{ data: null }`
- **Errors:** None (left side of Either is never used)
- **Events emitted:** none

### ListAuditLogs
- **Trigger:** Admin requests audit trail (GET /audit-logs)
- **Input:** filters (actorId?, entity?, action?, entityId?, startDate?, endDate?), cursor pagination params
- **Rules:** Results are ordered by creation date (newest first), cursor-paginated
- **Success:** `{ data: AuditLog[], meta: CursorPaginationMeta }`
- **Errors:** None
- **Events emitted:** none

## Cross-Domain Dependencies
- Depends on: `user` (by ID reference only — actorId)
- Subscribes to events from: `user` (UserCreatedEvent, UserLoggedInEvent, UserLoggedOutEvent, UserDeletedEvent, UserStatusChangedEvent)
- Emits events consumed by: none

## Authorization
- `CreateAuditLog`: internal only (called by event subscribers, not exposed via HTTP)
- `ListAuditLogs`: ADMIN only

## Constraints
- Append-only — no update or delete operations
- Cursor pagination — no offset pagination (audit logs grow unbounded)
- All date filters use inclusive range

## Edge Cases
- Event subscriber fails to create audit log — error is logged but does not block the original operation (fire-and-forget semantics)
- Actor is soft-deleted after audit log is created — actorId still references the deleted user (no cascade)
