# Supplier — Business Rules

## Responsibility

Supplier registry for tracking the origin of input purchases in the Inventory domain.

## Out of scope

- Price comparison across suppliers (post-MVP)
- Purchase history analytics and supplier performance (post-MVP)
- Purchase management (handled by Inventory domain)

## Entities

### Supplier

| Attribute | Type | Required | Rules |
|-----------|------|----------|-------|
| id | UUID | yes | generated by system |
| name | string | yes | unique, accent-insensitive comparison |
| phone | string | no | |
| email | string | no | valid email format when provided |
| status | enum | yes | ACTIVE or INACTIVE, defaults to ACTIVE |
| createdAt | DateTime | yes | set on creation |
| updatedAt | DateTime | yes | set on every update |
| deletedAt | DateTime | no | set on soft delete |

**Invariants:**
- Name must be unique across all active (non-soft-deleted) suppliers (accent-insensitive: "Agroquímica" and "Agroquimica" are the same)
- Cannot hard delete if any Purchase references this Supplier
- Soft delete when referenced by existing purchases

**Lifecycle:** ACTIVE ↔ INACTIVE (toggle). Soft-deleted when referenced and user requests deletion.

## Value Objects

- **Email** — optional, must be valid email format when provided (reuse existing Email value object from User domain if available)

## Use Cases

### CreateSupplier
- **Trigger:** user action
- **Input:** name, phone (optional), email (optional)
- **Rules:** name must be unique (accent-insensitive)
- **Success:** Supplier entity created
- **Errors:** SupplierAlreadyExistsError
- **Events emitted:** SupplierCreatedEvent

### EditSupplier
- **Trigger:** user action
- **Input:** supplierId, name (optional), phone (optional), email (optional)
- **Rules:** if name changed, new name must be unique (accent-insensitive)
- **Success:** Supplier entity updated
- **Errors:** SupplierNotFoundError, SupplierAlreadyExistsError
- **Events emitted:** SupplierUpdatedEvent

### DeleteSupplier
- **Trigger:** user action
- **Input:** supplierId
- **Rules:** hard delete if no purchases reference it. Soft delete if referenced.
- **Success:** Supplier deleted (hard or soft)
- **Errors:** SupplierNotFoundError
- **Events emitted:** SupplierDeletedEvent

### ToggleSupplierStatus
- **Trigger:** user action
- **Input:** supplierId
- **Rules:** toggles status between ACTIVE and INACTIVE
- **Success:** Supplier status toggled
- **Errors:** SupplierNotFoundError
- **Events emitted:** SupplierUpdatedEvent

### GetSupplier
- **Trigger:** user action
- **Input:** supplierId
- **Rules:** only return non-soft-deleted records
- **Success:** Supplier entity
- **Errors:** SupplierNotFoundError
- **Events emitted:** none

### ListSuppliers
- **Trigger:** user action or select dropdown (e.g., Purchase form)
- **Input:** pagination params (page, perPage), optional filters (status: ACTIVE/INACTIVE, search by name)
- **Rules:** only return non-soft-deleted records, paginated
- **Success:** paginated list of Supplier entities
- **Errors:** none
- **Events emitted:** none

## Cross-Domain Dependencies

- Depends on: none
- Referenced by: Inventory (Purchase references Supplier by ID)
- Emits events consumed by: Audit (all mutation events)

## Constraints

- Name uniqueness across all active (non-soft-deleted) suppliers, accent-insensitive
- Soft delete preserves data for audit trail and purchase history
- Hard delete only when no purchases reference the supplier
- All operations audited via domain events

## Edge Cases

- User submits "Agroquimica" when "Agroquímica" exists → reject with error suggesting existing name
- User deletes Supplier that has purchases → soft delete
- User deletes Supplier with no purchases → hard delete
- Inactive supplier shown in existing Purchase records but cannot be selected for new purchases
- User creates Supplier with name matching a soft-deleted Supplier → allowed (uniqueness only applies to active records)
- Soft-deleted suppliers remain visible in Purchase details (historical data preserved) but cannot be accessed directly or selected for new records
